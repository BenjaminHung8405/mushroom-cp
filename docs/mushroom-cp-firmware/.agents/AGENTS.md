# Workspace Rules

- Luôn tuân thủ nguyên tắc Poka-Yoke trong thiết kế và lập trình.
- Luôn luôn tuân thủ nguyên tắc LEAN trong lập trình.
- Luôn tìm root cause trước khi giải quyết vấn đề. Không bao giờ được suy đoán.
- Luôn ghi nhận tóm tắt những thay đổi vào file `CHANGELOG.md`, có chỉ dấu thời gian rõ ràng.
- Tạo test case khi có thể.
